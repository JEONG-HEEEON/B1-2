6️⃣ 핵심 기술 원리 — T2I vs I2V 선택 기준
T2I(Text-to-Image): 텍스트만으로 정지 이미지 생성 → 구도·조명·질감 통제 용이, 단 움직임 없음
I2V(Image-to-Video): 확정 이미지를 앵커로 모션만 추가 → 구도·톤 보존, 단 모션 통제는 상대적으로 어려움
선택 기준: 저렴한 T2I(Midjourney)로 4개 씬 구도·톤을 먼저 확정 → 모션이 서사에 결정적인 씬(2 위압감·3 변신 폭발·4 광선 액션)에만 I2V(Kling) 집중 적용. 씬1(정적 기상)·씬5(로고)는 정지 이미지 + CapCut 효과만으로 충분해 I2V 미적용, 크레딧 절약.

💡 차별점: 잔잔한 환경 모션 대신 **격렬한 액션 모션(폭발·변신·광선)**이 핵심이라 I2V 적용 씬을 늘려 배분.

7️⃣ 도구 선택 우선순위 — 제어 > 품질 > 비용 > 속도
제어(최우선): 12초 안에 B급 톤·캐릭터를 일관 각인 → 공통 프롬프트 접미사로 컬러·스타일 고정
품질(2순위): 제어 확보 후 액션 박력·이펙트 검수 채택 (씬3 재생성 사례)
비용(3순위): 사전 최적화 대신 Kling 크레딧 소진 시 Pika 전환으로 반응적 관리
속도(최하위): 이미지 톤 확정 후에만 영상 변환 → 재생성 최소화(비용 절감) 우선
8️⃣ 캐릭터/스타일 일관성 제어 방법
도구의 Character Reference/Seed 고정 기능은 미사용. 대신 두 가지로 통제:

프롬프트 레벨 통제: 공통 접미사로 컬러·스타일 통일
캐릭터 특징 고정 서술: 주인공(반곱슬·안경·다크서클·구겨진 셔츠), 괴물(빨간 눈·이빨·거대 알람시계 본체)의 핵심 특징을 매 씬 반복 명시 → 씬3 톤 이탈 시 원칙에 따라 재생성
9️⃣ 심층 인터뷰 응답
9-1. 도구 결과물 간 불일치 보정 기준과 절차
씬 생성 시마다 직전 씬과 나란히 놓고 (a) 컬러(레드/블랙 하이 콘트라스트 유지), (b) 캐릭터 형상 일관성(주인공·괴물 특징), (c) B급 코믹 톤 유지 여부 3가지를 육안 검수했습니다. 컬러가 어긋나면 프롬프트에 matching high-contrast red and black tone as previous scene을 추가해 재생성, 톤이 진지하게 나온 씬3은 comic, over-the-top 강조 후 전면 재생성했습니다. 해상도 불일치는 생성 단계가 아닌 CapCut 캔버스 1080p 통일 설정으로 편집 단계에서 일괄 정규화했습니다.

9-2. 다른 도구를 사용했다면 예상되는 차이

이미지: Midjourney → DALL-E 교체 시, DALL-E는 프롬프트 해석의 직관성은 높으나 과장된 코믹 액션·역동적 구도 표현이 Midjourney보다 다소 정적으로 나올 가능성이 있어, B급 톤을 살리려면 공통 접미사에 exaggerated dynamic action pose를 추가 튜닝해야 했을 것으로 예상됩니다.
비디오: Kling → Pika 교체 시, Pika는 짧은 클립·환경 모션에는 강하나 폭발·변신 같은 강한 이펙트 모션의 박력이 약해질 수 있어, 씬3·4의 임팩트를 자막·효과음으로 보강해야 했을 것으로 예상됩니다.
